Раздел 02

Проблема рынка

Ключевые тезисы

  • Разрозненные базы моделей у разных агентств Каждое агентство ведёт свою таблицу — данные не пересекаются, нет единого каталога
  • Ручной подбор через мессенджеры и таблицы WhatsApp, Telegram, Excel — хаотичная коммуникация, потеря информации
  • Нет прозрачности процесса для клиента Клиент не видит статус заказа, не может отслеживать подбор
  • Потеря времени на координацию между участниками Кастинг-директор, агент, модель, клиент — все общаются вручную

Раздел 10

Заказ глазами клиента

Ключевые тезисы

  • Клиент видит только моделей, подтвердивших участие Внутренняя кухня (отказы, переговоры с агентствами) скрыта
  • Прогресс заказа по фазам в реальном времени Визуальная шкала: сколько моделей на каждом этапе из заказанных
  • Анонимные композитки до определённой фазы Имена и агентства раскрываются только после подтверждения
  • Избранное по ходу работы с заказом Клиент помечает понравившихся моделей для будущих заказов

Раздел 13

Безопасность и приватность

Ключевые тезисы

  • Соответствие ФЗ-152 (персональные данные) Согласие на обработку, экспорт данных, право на удаление (30 дней)
  • JWT аутентификация + CSRF защита Токен в httpOnly cookie — не доступен из JavaScript
  • Защита от парсинга базы (anti-scraping, watermarks) 9 уровней защиты: rate limit, CAPTCHA, fingerprinting, водяные знаки
  • Подписанные URL для фото с TTL Ссылка на фото действует 15 минут, потом истекает
  • Аудит лог всех действий Кто, когда, что сделал — полная трассировка
